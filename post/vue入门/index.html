vuevue入门

vue、angular、react要解决的问题是：数据驱动，避免手动的操作DOM。

vue特点：

  1. 更加轻量 20kb
  2. 渐进式框架（不需要学习完vue的所有知识便可以应用到系统中）
  3. 响应式的更新机制
  4. 学习成本低

v-bind

v-bind:id = "message"

此时id属性会根据后台message数据的值动态变换。上面指令可以简写为：

:id = "message"

插槽slot

上面的组件1和组件2，组件2引用了组件1。但是组件1直接以字符串形式在组件2中写死了。如果组件1进行调整的话还得到引用它的组件中进行修改。于是vue引入了slot。

Vue.component("todo-list",{
    template:
    ` <ul>
      <slot></slot>
    </ul>`
    });

组件使用如下：

 <todo-list>
   <todo-item v-for="item in items" :title="item.title" :del="item.del" />
 </todo-list>

如果需要引入多个slot时，在组件定义时还要给slot加上name属性。

在组件使用时：

2.5加上slot属性，值为定义其时的name属性。

2.6

<template v-slot:slot定义的name属性>
		xxxxxxxxx
</template>

Vue.Component缺点

  1. 全局定义：强制要求每个component中的命名不得重复
  2. 字符串模版：缺乏语法高亮，在HTML有多行的时候，需要用到丑陋的
  3. 不支持css：意味着当HTML和javascript组件化时，CSS明显被遗漏
  4. 没有构建步骤：限制只能使用HTML和ES5 Javascript，而不能使用预处理器，如Babel。
